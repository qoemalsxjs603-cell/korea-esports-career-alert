# 맞춤형 e스포츠 채용 공고 수집기 v9

v8에서 확인된 중복·오탐·누락을 해결하기 위해 사이트별 전용 파서를 추가한 버전입니다.

## 핵심 수정

### 충남콘텐츠진흥원
- 제목이 아니라 URL의 `wr_id`를 공고 고유번호로 사용합니다.
- `N`, `새글` 표시가 사라져도 같은 공고로 인식합니다.
- `진행중` 상태의 실제 직원채용 공고만 감지합니다.
- 합격·서류전형·면접전형·결과 공고를 제외합니다.

### 디플러스 기아 / 잡코리아
- `/Recruit/GI_Read/숫자` 형식의 실제 공고 링크만 수집합니다.
- JOBKOREA, 전체 채용정보, 신입공채, 헤드헌팅 등 메뉴 링크를 제외합니다.
- 상세페이지와 마감일을 확인하여 마감·종료된 과거 공고를 제외합니다.
- 상세페이지를 읽지 못하고 활성 상태도 확인되지 않으면 알림을 보류합니다.

### T1
- 링크를 찾는 방식이 아니라 페이지 본문을 `[Esports T1]`, `[esports T1 Academy]` 제목 단위로 분리합니다.
- 각 공고의 접수 시작일과 마감일을 추출합니다.
- 마감된 공고는 제외합니다.
- `공고 제목 + 접수 시작일`을 고유번호로 사용하므로 동일 직무가 재공고되면 새 공고로 감지합니다.

### 전체 사이트
- `seen_ever`에 한 번 본 공고 ID를 영구 저장합니다.
- 사이트가 잠시 0건으로 로딩되어도 기존 상태를 지우지 않습니다.
- 공고 수가 비정상적으로 급감하면 보호 장치를 작동합니다.
- 메일에 게시일·마감일·최초 발견 시각을 표시합니다.
- `data/last_run_report.json`에서 사이트별 수집 결과를 확인할 수 있습니다.

## 첫 v9 실행

`config/settings.json`의 다음 설정 때문에 T1만 현재 활성 공고를 최초 1회 메일로 보냅니다.

```json
"notify_existing_on_first_run_sources": ["t1"]
```

다른 사이트의 기존 공고는 기준값으로만 저장합니다.

T1의 현재 공고도 메일로 받고 싶지 않다면 위 값을 다음처럼 바꾸세요.

```json
"notify_existing_on_first_run_sources": []
```

## Actions 실행 방식

- `normal`: 정상 수집 및 새 공고 메일 발송
- `test_email`: 사이트를 수집하지 않고 테스트 메일만 발송
- `diagnostic`: 수집하지만 메일과 실제 기준값은 건드리지 않고 보고서만 저장
- `reset_baseline`: 모든 사이트의 현재 공고를 새 기준값으로 저장하고 메일은 보내지 않음

## 중요

v8의 잘못 저장된 공고 이력이 남지 않도록 `data/state.json`을 반드시 v9 파일로 덮어써야 합니다.
