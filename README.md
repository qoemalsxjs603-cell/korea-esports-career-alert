# 맞춤형 e스포츠 채용 공고 수집기 v10

이번 버전은 이전 버전의 부분 수정이 아니라 구조를 새로 만든 최종 안정화판입니다.

## 핵심 원칙

1. T1·Gen.G·농심·충남콘텐츠진흥원·잡코리아는 각각 전용 파서를 사용합니다.
2. 실제 공고 구조가 검증된 경우에만 상태를 변경하고 메일을 보냅니다.
3. 사이트가 차단되거나 구조를 확인하지 못하면 0건으로 처리하지 않고 기존 상태를 보존합니다.
4. 첫 v10 normal 실행은 기존 공고를 기준값으로만 저장하며 메일을 보내지 않습니다.
5. 이후 처음 나타난 고유 공고 ID만 메일로 보냅니다.
6. diagnostic은 메일과 state.json을 건드리지 않습니다.

## 적용 순서

1. ZIP의 모든 파일로 기존 저장소를 교체합니다.
2. GitHub Secrets는 기존 것을 그대로 사용합니다.
3. Actions에서 `diagnostic`을 실행합니다.
4. `data/last_run_report.json`을 확인합니다.
5. 이어서 `normal`을 한 번 실행합니다. 이 실행은 v10 기준값 생성입니다.
6. 이후 2시간마다 자동 실행되며 검증된 새 공고만 메일로 옵니다.

## 실행 옵션

- `diagnostic`: 수집 결과만 확인, 상태·메일 변경 없음
- `normal`: 정상 실행
- `baseline`: 현재 공고로 기준값 재생성
- `test_email`: 메일 연결 확인
- `notify_t1_current`: 현재 T1 활성 공고를 한 번 메일로 받기

## 기대되는 핵심 진단

- 충남콘텐츠진흥원: `wr_id` 기준으로 한 공고당 한 ID
- 디플러스 기아: 메뉴·과거 마감 공고 제외
- T1: 접수기간이 지난 공고 제외, 날짜 표시
- Gen.G: 메뉴와 인재풀 제외, 실제 `/o/공고번호`만 표시
- 농심: 현재 공고가 없으면 명시적인 0건으로 검증
