# 맞춤형 e스포츠 채용 공고 수집기 v8

사용자가 지정한 **14개 채용 페이지만** 2시간마다 확인합니다.  
사람인·잡코리아 전체 검색이나 광범위한 키워드 검색은 하지 않습니다.

- 스마일게이트: `IP이스포츠`, `이스포츠`

- 충남콘텐츠진흥원: `직원채용`

## 핵심 동작

1. GitHub Actions가 2시간마다 실행됩니다.
2. 처음 실행할 때 보이는 기존 공고는 **기준값으로만 저장**합니다.
3. 이후 새로 나타난 공고만 이메일로 보냅니다.
4. 공고가 없으면 메일을 보내지 않습니다.
5. 수집 실패한 사이트는 기존 기준값을 지우지 않습니다.
6. 같은 공고가 여러 DOM 요소에서 잡혀도 URL 기준으로 중복 제거합니다.

## 설치

1. 새 GitHub 저장소를 만들고 ZIP의 파일을 모두 업로드합니다.
2. 저장소 `Settings → Secrets and variables → Actions`에서 아래 Secrets를 확인합니다.

권장 이름:

- `SMTP_USER`: 발송용 Gmail 주소
- `SMTP_PASSWORD`: Gmail 앱 비밀번호
- `EMAIL_TO`: 수신 이메일 주소

기존 저장소가 `EMAIL_USER`, `EMAIL_PASSWORD`, `GMAIL_USER`,
`GMAIL_APP_PASSWORD`, `MAIL_TO`를 사용하던 경우에도 호환됩니다.

`EMAIL_TO`를 설정하지 않으면 `config/settings.json`의 수신 주소를 사용합니다.

3. `Actions` 탭에서 **Focused Esports Job Alert**를 열고
   `Run workflow → test_email`을 한 번 실행합니다.
4. 테스트가 정상이라면 `Run workflow → normal`을 실행합니다.
   최초 normal 실행은 기준값만 생성하므로 실제 공고 메일은 오지 않는 것이 정상입니다.

## 기존 프로젝트를 교체할 때

기존 저장소의 파일을 모두 지우고 이 ZIP 내용을 업로드하는 방식이 가장 안전합니다.
다만 GitHub Secrets는 저장소에 그대로 남으므로 다시 만들 필요가 없습니다.

기존 워크플로 파일이 남아 있으면 예전 수집기가 함께 실행될 수 있으므로
`.github/workflows` 폴더에는 `job-alert.yml` 하나만 남겨주세요.

## 키워드·링크 수정

`config/sources.json`에서 변경합니다.

- `"mode": "keyword"`: 지정 키워드가 포함된 공고만 감지
- `"mode": "new"`: 해당 페이지의 새 공고 전부 감지
- `"mode": "criteria"`: 지역·근무형태 조건을 모두 통과한 공고만 감지

키워드는 공백과 하이픈 차이를 무시합니다. 예를 들어 `e-스포츠`,
`e 스포츠`의 표기 차이로 인한 누락을 줄였습니다.

## 주의

- LinkedIn은 로그인 유도나 접속 제한으로 자동 수집이 간헐적으로 실패할 수 있습니다.
- 사이트가 HTML 구조를 크게 변경하면 해당 사이트의 추출 규칙 조정이 필요할 수 있습니다.
- GitHub Actions의 예약 실행은 서버 상황에 따라 수 분 정도 늦어질 수 있습니다.
