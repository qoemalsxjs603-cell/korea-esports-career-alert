V9.1 교체 순서

1. 기존 GitHub 저장소의 Code 화면으로 이동합니다.
2. 아래 파일을 v9 파일로 덮어씁니다.
   - job_alert.py
   - requirements.txt
   - README.md
   - config/settings.json
   - config/sources.json
   - data/state.json
   - data/last_run_report.json
3. .github/workflows/job-alert.yml 파일을 열어 연필(Edit) 버튼을 누릅니다.
4. 기존 내용을 전부 지우고 v9의 job-alert.yml 내용을 붙여넣습니다.
5. Commit changes를 누릅니다.
6. Actions → Focused Esports Job Alert v9 → Run workflow로 이동합니다.
7. 먼저 diagnostic을 실행합니다. diagnostic은 실제 기준값을 바꾸지 않습니다.
8. 실행이 초록색 Success이면 data/last_run_report.json을 열어 T1, 충남콘텐츠진흥원, 디플러스 기아 결과를 확인합니다.
9. 이상이 없으면 normal을 실행합니다. 첫 normal 실행에서는 T1의 현재 활성 공고만 최초 1회 알림되고, 다른 사이트는 기준값만 저장됩니다.
10. 이후 2시간마다 자동 실행됩니다.

주의:
- data/state.json을 덮어쓰지 않으면 v8의 잘못된 중복 기록이 남을 수 있습니다.
- 기존 workflow 파일을 새 파일로 추가하지 말고, 같은 job-alert.yml 파일의 내용을 교체하세요.
- GitHub Secrets는 기존 저장소에 그대로 남으므로 다시 만들 필요가 없습니다.
